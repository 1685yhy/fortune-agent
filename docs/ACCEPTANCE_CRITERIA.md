# v3.0 验收标准 — Life Curation 品牌升级

## 总体原则
- 每个 Sprint 完成后必须通过所有验收条件
- 测试驱动：先写验收条件，再写代码
- 每完成一个 Sprint 推一次 GitHub
- 定期用 design-critique 技能审查

---

## Sprint 1: AI 人格重塑（底层地基）

### S1.1 语气体系
- [ ] 移除所有「小友」「老夫」等老旧称呼
- [ ] 实现 3 种 AI 人格模式：
  - 毒舌闺蜜（默认）：犀利、有趣、能玩梗
  - 理性分析师：专业、严谨、数据化
  - 温柔陪伴者：温暖、共情、鼓励
- [ ] 用户可说「换个风格」切换人格
- [ ] 每种人格有独立的 System Prompt

### S1.2 情绪安抚优先
- [ ] 收到任何请求时，先返回情绪安抚语句（< 1 秒）
- [ ] 安抚内容包括：共情 + 正常化 + 赋能引导
- [ ] 不安抚直接给答案 → 验收失败

### 验收方式
```python
# 1. 发送"我该不该跳槽，纠结半年了"→ 检查回复是否先安抚
# 2. 发送"换个风格"→ 检查人格是否切换
# 3. 发送 10 条不同消息 → 检查无"小友""老夫"
```

## Sprint 2: 概率化表达 + 行动建议

### S2.1 概率化
- [ ] 所有预测必须用概率表达（"约60%可能性"而非"一定会"）
- [ ] 负面预测必须附带「如果注意XX，可以降低风险」
- [ ] 明确标注哪些是「高概率」哪些是「仅供参考」

### S2.2 行动建议
- [ ] 每条分析后附带 3-5 条具体行动建议
- [ ] 建议格式：具体行动 + 时间节点 + 成功指标
- [ ] 建议来源标注（基于用神/基于格局/基于流月）

### 验收方式
```python
# 1. 发送八字查询 → 检查回复是否含概率词
# 2. 检查行动建议是否包含：行动+时间+指标
# 3. 发送 20 个案例 → 检查建议多样性（不重复）
```

## Sprint 3: 视觉重设计

### S3.1 品牌调性
- [ ] 色彩体系：黑白金为主，五行色点缀
- [ ] 字体：现代无衬线 + 中文优化
- [ ] 所有图表（八字/紫微/风水）统一风格

### S3.2 杂志级报告
- [ ] 聊天回复改为「卡片式」布局
- [ ] 命盘图包含：图表 + 关键数字 + 一句金句
- [ ] 分享卡片：精简版 + 二维码

### 验收方式
```python
# 1. 生成 10 张命盘图 → 用 visual-audit 审查
# 2. 用 color-systems 审查色彩一致性
# 3. 用 craft-signals 审查细节质量
```

## Sprint 4: 复购引擎

### S4.1 命运追踪
- [ ] 用户可以查看历史预测
- [ ] 显示准确率（"过去3个月财运预测准确率85%"）
- [ ] 每周推送更新的运势

### S4.2 付费优化
- [ ] 免费版：每天 1 次基础问答
- [ ] 会员版：¥39.9/月，无限 + 行动建议 + 分享卡片
- [ ] 深度报告：¥99/次，完整 PDF 报告

### 验收方式
```python
# 1. 查询历史预测 → 检查是否显示准确率
# 2. 模拟 100 用户 × 30 天 → 检查推送到达率
# 3. 测试购买流程 → 检查支付成功率
```
