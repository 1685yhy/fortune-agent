# 真实对话模拟测试 + 数据质量验证 — 验收标准

## 原则
> 测试必须模拟「真人在微信里怎么聊天」，不是单句问答。
> 数据入库前必须验证：来源可靠、内容准确、格式规范。

---

## S1: 多轮对话场景库

### 验收标准
- [ ] S1.1 构建 ≥ 20 个多轮对话场景（每个 3-8 轮）
- [ ] S1.2 场景覆盖全部 11 种命理类型 + 心事树洞 + 日历
- [ ] S1.3 每个场景包含：用户画像、对话历史、每轮预期行为、通过标准
- [ ] S1.4 场景来源：真实社交平台用户对话模式（非编造）
- [ ] S1.5 包含边界场景：错别字、语音转文字误差、emoji 轰炸、超长消息

### 场景模板
```
场景: 焦虑妈妈问孩子学业
用户画像: 35岁女性，孩子上初中，焦虑孩子成绩
对话:
  轮1: "老师你好，我儿子最近成绩下滑很厉害，我想看看他八字"
  轮2: "2012年8月15日 上午10点 郑州 男"
  轮3: "他适合学文科还是理科？"
  轮4: "那以后能考上好大学吗？"
预期: 轮1引导提供信息 → 轮2分析学业运 → 轮3关联八字给出方向 → 轮4结合大运给趋势
```

---

## S2: 对话模拟引擎

### 验收标准
- [ ] S2.1 自动执行多轮对话（模拟用户发送消息→接收回复→发下一轮）
- [ ] S2.2 每轮检查是否满足预期行为（非简单关键词匹配）
- [ ] S2.3 记录：响应时间、字数、是否索要信息、情绪质量、上下文引用
- [ ] S2.4 生成测试报告：通过率、失败详情、性能统计
- [ ] S2.5 总通过率 ≥ 80%

---

## S3: 数据质量验证

### 验收标准
- [ ] S3.1 每条爬取数据必须通过 3 道验证才能入库
- [ ] S3.2 验证1：来源可信（权威网站/古籍/学术来源，排除 SEO 垃圾站）
- [ ] S3.3 验证2：内容完整（≥100字，包含实质信息，非标题党）
- [ ] S3.4 验证3：去重检查（与已有数据相似度 < 80%）
- [ ] S3.5 验证失败的条目记录到 rejected.jsonl 并标注原因
- [ ] S3.6 新增目标：八字案例 ≥ 500条，风水实操 ≥ 300条，面相知识 ≥ 200条

---

## S4: 数据清洗管道

### 验收标准
- [ ] S4.1 自动去除 HTML 标签、广告、导航、页脚
- [ ] S4.2 统一编码 UTF-8，统一换行符
- [ ] S4.3 分段合理（每条 200-800 字，保持语义完整）
- [ ] S4.4 标注数据来源 URL 和爬取时间
- [ ] S4.5 清洗后数据存入 staging/ 目录，人工抽检后移入 production/
