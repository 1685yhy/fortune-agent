# 微信小程序 + 公众号 — 验收标准

## 原则
> 后端不变，前端适配。一套 API 服务所有渠道。
> 小程序核心体验：拍照面相 → 秒出分析 → 分享朋友圈。

---

## M1: 公众号接入

### 验收标准
- [ ] M1.1 CoW 配置公众号通道（channel_type: wechatmp）
- [ ] M1.2 公众号 Token 验证通过
- [ ] M1.3 公众号发送文字消息 → 正常对话
- [ ] M1.4 公众号发送图片 → 面相/手相分析
- [ ] M1.5 公众号菜单配置（今日运势/八字分析/面相分析）
- [ ] M1.6 公众号关注回复 → 欢迎语 + 引导设置八字

---

## M2: 小程序架构

### 页面规划（5 个页面）
```
pages/
  ├── index/          # 首页 — 今日运势卡片 + 快捷入口
  ├── chat/           # 对话 — AI 对话界面
  ├── face/           # 面相 — 拍照/选照片 → 分析结果
  ├── calendar/       # 日历 — 7 天运势预览
  └── profile/        # 我的 — 仪表盘 + 设置
```

### 验收标准
- [ ] M2.1 首页展示今日运势卡片（调用 /api/calendar/daily）
- [ ] M2.2 对话页完整 AI 聊天功能（调用 /api/chat）
- [ ] M2.3 面相页拍照/上传 → 分析结果展示（调用 /api/face-reading）
- [ ] M2.4 日历页 7 天预览 + 每日详情
- [ ] M2.5 个人页展示仪表盘数据（调用 /api/dashboard/{id}）
- [ ] M2.6 所有页面 < 3s 加载
- [ ] M2.7 支持微信登录（wx.login → 服务端换取 user_id）

---

## M3: 多通道统一

### 验收标准
- [ ] M3.1 CoW 同时运行企业微信 + 公众号 + 微信通道
- [ ] M3.2 同一 user_id 跨渠道数据互通（偏好/历史/八字）
- [ ] M3.3 小程序通过 API Key 鉴权访问后端
- [ ] M3.4 各渠道错误隔离（一个渠道挂了不影响其他）

---

```
                    │
         ┌──────────┴──────────┐
```

## 执行顺序

```
M1 (公众号) → M4 (API文档) → M2 (小程序) → M3 (统一)
```
