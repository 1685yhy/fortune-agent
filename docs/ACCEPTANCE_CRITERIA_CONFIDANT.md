# 心事树洞 — 验收标准

## 原则
> 用户来算命不是为了「被算」，而是为了「被理解」。
> 70% 的用户带着故事来，我们要先听完，再温柔地问「要不要从命理角度看看？」

---

## H1: 倾诉检测

### 验收标准
- [ ] H1.1 用户发送故事型消息（70-200字，包含情绪+经历）→ 识别为「倾诉模式」
- [ ] H1.2 倾诉模式 ≠ 命理请求（「帮我看看八字」不触发倾诉模式）
- [ ] H1.3 检测在 MessageAnalyzer 中完成（不增加额外 API 调用）
- [ ] H1.4 测试 10 条真实用户倾诉 → 正确识别率 > 80%

### 验收测试
```python
# 这些应该触发倾诉模式
assert is_sharing("最近工作压力好大，老板总刁难我，不知道要不要辞职")
assert is_sharing("分手三个月了还是走不出来，每天都会想起他")
assert is_sharing("我觉得自己运气一直很差，做什么都不顺")

# 这些不应该触发
assert not is_sharing("帮我看看八字")
assert not is_sharing("今日运势")
assert not is_sharing("你好")
```

---

## H2: 深度倾听

### 验收标准
- [ ] H2.1 倾诉模式下，AI 先表达共情（不套模板，针对用户说的内容）
- [ ] H2.2 AI 问 1-2 个开放式追问（「当时发生了什么？」「你现在的感受是？」）
- [ ] H2.3 连续 2-3 轮倾听后，AI 自然过渡：「想不想从命理角度看看？」
- [ ] H2.4 倾听阶段绝对不索要出生信息
- [ ] H2.5 倾听回复速度 < 5 秒（使用 Flash）

---

## H3: 自然过渡

### 验收标准
- [ ] H3.1 过渡语由 AI 动态生成（不硬编码「要不要看看八字」）
- [ ] H3.2 过渡语包含：对用户处境的呼应 + 命理能提供什么帮助的说明
- [ ] H3.3 用户说「好」→ 进入正常八字收集流程
- [ ] H3.4 用户说「不用了」→ 继续倾听，不强制
- [ ] H3.5 过渡语匹配用户人格风格

---

## H4: 集成

### 验收标准
- [ ] H4.1 倾诉模式无缝集成到现有 process() 流程
- [ ] H4.2 不影响现有命理分析功能
- [ ] H4.3 8 场景回归测试全部通过
- [ ] H4.4 3 条真实倾诉场景测试全部通过
