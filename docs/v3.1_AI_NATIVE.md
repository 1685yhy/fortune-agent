# v3.1 验收标准 — AI 原生架构升级

## 核心原则
> 能用 AI 动态生成的，绝不用硬编码规则。
> 每个模块在给出答案前，先问：这个回答是「千人一面」还是「千人千面」？

---

## Sprint 5: 自适应 AI 行动建议

### 目标
把硬编码的建议规则表替换为 AI 动态生成，基于用户完整命盘 + 当前处境。

### 验收标准
- [ ] S5.1 用户发送八字 → 行动建议由 LLM 实时生成（不再使用 advisor.py 硬编码表）
- [ ] S5.2 行动建议必须包含：具体行动 + 时间窗口 + 预期效果
- [ ] S5.3 同一八字不同处境 → 生成不同的建议（验证千人千面）
- [ ] S5.4 建议风格匹配当前人格模式（毒舌/分析/温柔）
- [ ] S5.5 测试：10 组不同八字 → 10 组不同建议，零重复率 > 80%

### 验收测试
```python
# 同一八字，不同处境 → 不同建议
r1 = analyze("八字A", context="想跳槽")
r2 = analyze("八字A", context="想谈恋爱")
assert r1 != r2  # 必须不同

# 10 组八字 → 检查建议多样性
for 10 samples:
    advice = generate(bazi)
    assert len(set(advice)) > 8  # >80% 不重复
```

## Sprint 6: 名人对照引擎

### 目标
从 1005 人名人库中找到与用户命盘最相似的，生成对比洞察。

### 验收标准
- [ ] S6.1 输入用户八字 → 在 1005 人中找到 TOP3 最相似命盘
- [ ] S6.2 相似度计算：日柱相似 + 格局相同 + 用神相近
- [ ] S6.3 AI 生成对比洞察：「你和马云的格局相同，都是七杀格身弱。他当年也被说疯子，后来...」
- [ ] S6.4 输出包含：相似度评分 + 关键差异 + 人生启示
- [ ] S6.5 响应时间 < 3 秒（1005 人匹配）

### 验收测试
```python
r1 = match(bazi_1990)
r2 = match(bazi_1984)
assert r1 != r2  # 不同八字匹配不同名人
assert r1["top_match"]["score"] > 0.5  # 有一定相似度
assert "insight" in r1["top_match"]  # AI 生成洞察
```

## Sprint 7: 自适应人格

### 目标
AI 根据用户消息自动检测情绪，选择合适的语气风格，不再写死 3 种 Prompt。

### 验收标准
- [ ] S7.1 用户消息包含焦虑/迷茫 → 自动切温柔陪伴模式
- [ ] S7.2 用户消息包含调侃/幽默 → 自动切毒舌闺蜜模式
- [ ] S7.3 用户消息包含数据/分析需求 → 自动切理性分析师模式
- [ ] S7.4 用户可手动覆盖：「今天想听毒舌的」
- [ ] S7.5 情绪检测准确率 > 80%（用 50 条标注数据测试）

### 验收测试
```python
assert detect_mood("我好纠结要不要分手") == "gentle"
assert detect_mood("帮我看看这个投资划不划算") == "analyst"
assert detect_mood("哈哈哈今天心情超好") == "sassy"
```

## Sprint 8: AI 驱动幸运日历

### 目标
基于用户八字 + 流日流月，AI 动态生成个性化「今日宜忌」。

### 验收标准
- [ ] S8.1 输入八字 + 日期 → 生成当天专属宜忌（不少于 3 条宜、3 条忌）
- [ ] S8.2 每条宜忌包含：事项 + 最佳时间窗口 + 原因（基于流日）
- [ ] S8.3 日历时自动适配用户人格风格
- [ ] S8.4 支持未来 7 天预览
- [ ] S8.5 不同用户同日 → 不同建议（千人千面）

### 验收测试
```python
cal1 = generate_calendar(bazi_A, "2026-07-20")
cal2 = generate_calendar(bazi_B, "2026-07-20")
assert cal1 != cal2  # 同一天不同用户的建议不同
assert len(cal1["宜"]) >= 3
assert "时间" in cal1["宜"][0]  # 有时机建议
```

## Sprint 9: AI 驱动智能图表

### 目标
AI 决定图表展示什么、突出什么、用什么颜色风格。

### 验收标准
- [ ] S9.1 AI 根据命盘特点决定图表高亮项（如缺木→突出木相关建议）
- [ ] S9.2 AI 生成一句个性化金句放在图表上
- [ ] S9.3 分享卡片包含 AI 选的最佳配色 + 布局
- [ ] S9.4 同一命盘生成 3 张不同风格的图表，用户可选

## 执行要求

- 每个 Sprint 独立验收
- 每完成一个 Sprint：测试 → 部署 → GitHub
- 用 design-critique 技能审查每个 Sprint 的质量
- 禁止硬编码：新增代码中 hardcoded strings < 10 条
