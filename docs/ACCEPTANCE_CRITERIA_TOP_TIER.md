# 顶级水平打磨 — 验收标准

## 原则
> 不是「能用就行」，是「用户想不到可以这么好」。
> 每个模块验收前问：这个体验在市面上排第几？

---

## D1: 数据扩充至 50K（影响：最大）

### 验收标准
- [ ] D1.1 知识库从 18K → ≥ 50,000 条验证数据
- [ ] D1.2 新增数据源 ≥ 5 个（优先古籍、优先实操案例）
- [ ] D1.3 八字案例 ≥ 5,000 条（含名人命例 + 民间案例）
- [ ] D1.4 风水实操 ≥ 3,000 条（含户型图分析 + 布局指导）
- [ ] D1.5 梦境解析 ≥ 10,000 条（周公解梦 + 现代心理学解读）
- [ ] D1.6 手相知识 ≥ 1,000 条（掌纹 + 指型 + 掌色）
- [ ] D1.7 奇门遁甲 ≥ 2,000 条（古籍原文 + 白话解读）
- [ ] D1.8 100% 条目通过 3 道验证（来源→内容→去重）
- [ ] D1.9 拒收率 < 15%（爬虫自带初筛）
- [ ] D1.10 入库后 RAG 检索 Top-5 命中率 > 80%（人工抽检 100 条）

### 验证流程
```
爬取 → 初筛(长度>80字) → 验证1(来源可信) → 验证2(内容完整) → 验证3(去重) → 入库 → 抽检100条
```

## D2: 速度优化（影响：大）

### 验收标准
- [ ] D2.1 高频问题缓存命中 < 1s（今日运势/八字格式/帮助/问候）
- [ ] D2.2 缓存命中率 > 30%（30% 的请求从缓存返回）
- [ ] D2.3 流式响应：Pro 分析首字节 < 5s（用户看到「正在分析」后开始输出）
- [ ] D2.4 图表缓存：同八字 24h 内直接返回已有图片（不重渲染）
- [ ] D2.5 Flash 回答 < 5s P95（不含 Pro 分析）
- [ ] D2.6 Pro 超时兜底：>90s → Flash 回退（已有，需验证）

### 验收测试
```python
# 缓存命中测试
r1 = ask("今日运势")  # 冷启动 ~3s
r2 = ask("今日运势")  # 缓存命中 < 1s
assert r1 == r2

# 流式响应测试
for chunk in stream_analyze("八字..."):
    assert chunk  # 每个 chunk 非空
    assert total_time < 5  # 首字 < 5s
```

## D3: 主动关怀（影响：中）

### 验收标准
- [ ] D3.1 每日早安推送：早 7:30 推今日运势卡片（含宜忌+幸运+一句话）
- [ ] D3.2 推送使用用户偏好：风格/话题/长度适配
- [ ] D3.3 重要节点提醒：生日前 3 天提醒、换大运提醒、冲煞日提醒
- [ ] D3.4 情绪回访：用户在树洞倾诉后 2-3 天，推送「最近好点了吗？」
- [ ] D3.5 个人命理仪表盘：总览页面（八字概览+最近分析+准确率+偏好）
- [ ] D3.6 精美分享卡片：生成可发朋友圈的运势卡片（混合图表+文案）
- [ ] D3.7 推送不重复、不骚扰（同一用户每天最多 1 条推送）

---

## D4: 多模态完善（影响：中）

### 验收标准
- [ ] D4.1 手相分析：上传手掌照片 → 掌纹检测 → 生命线/智慧线/感情线分析
- [ ] D4.2 风水照片：上传房间照片 → 空间识别 → 家具布局分析 + 改善建议
- [ ] D4.3 手相准确率：手掌检测 > 95%，主线识别 > 80%
- [ ] D4.4 风水识别：门窗/家具检测 > 85%
- [ ] D4.5 面相引擎优化：真图回测 50 张 → 脸型准确率 > 85%
- [ ] D4.6 全部使用本地 CV（不依赖外部视觉 API）

---

## 执行顺序

```
D1 (数据50K) → D2 (速度) → D3 (主动关怀) → D4 (多模态)
   ↑                ↑              ↑               ↑
 最大影响          大影响         中影响          中影响
 爬虫后台跑        缓存+流式      推送+仪表盘      CV扩展
```

### 每个 D 级完成后
1. 跑 8 场景回归测试
2. 用 20 场景模拟测试
3. 服务器部署验证
4. GitHub 提交 + push
