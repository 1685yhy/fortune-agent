# AI 幸运日历 — 验收标准

## 原则
> 每个用户每天看到的日历都不同。基于真实八字流日计算，AI 动态生成。
> 这不是黄历 App —— 这是你的专属每日运势管家。

---

## C1: 日历引擎

### 验收标准
- [ ] C1.1 输入用户八字 + 日期 → 输出当天个性化宜忌
- [ ] C1.2 宜 ≥ 3 条，忌 ≥ 3 条（AI 生成，不做硬编码）
- [ ] C1.3 每条包含：事项 + 最佳时间窗口（时辰）+ 命理原因（基于流日干支）
- [ ] C1.4 不同八字同一日期 → 内容不同（验证千人千面）
- [ ] C1.5 生成速度 < 5 秒（单日 Flash 调用）

### 验收测试
```python
cal1 = generate_calendar(bazi_A, "2026-07-20")
cal2 = generate_calendar(bazi_B, "2026-07-20")
assert cal1["宜"] != cal2["宜"]
assert len(cal1["宜"]) >= 3
assert "时间" in cal1["宜"][0]
```

---

## C2: 7 天预览

### 验收标准
- [ ] C2.1 支持未来 7 天预览（今日 + 6 天）
- [ ] C2.2 每天独立生成（不复制前一天）
- [ ] C2.3 标注「重点日」（如冲煞、三合、六合日）
- [ ] C2.4 生成速度 < 15 秒

---

## C3: 人格适配

### 验收标准
- [ ] C3.1 毒舌模式日历 → 犀利调侃风格（「今天别作死投资」）
- [ ] C3.2 分析师模式日历 → 数据化呈现（「财运指数 78%」）
- [ ] C3.3 温柔模式日历 → 温暖鼓励风格（「今天适合给自己放个小假」）
- [ ] C3.4 自动读取用户偏好人格（从反馈闭环学到的）

---

## C4: API + 推送

### 验收标准
- [ ] C4.1 POST /api/calendar/daily — 今日日历（需 user_id 或八字）
- [ ] C4.2 POST /api/calendar/week — 7 天预览
- [ ] C4.3 已保存八字的用户只需传 user_id，系统自动读取
- [ ] C4.4 新用户（无八字）→ 返回通用黄历 + 引导设置八字

2. 每级完成：部署 → 测试 → 8 场景回归
