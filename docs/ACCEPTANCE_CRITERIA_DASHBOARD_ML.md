# 仪表盘 + 流式 + ML 质量 — 验收标准

## 原则
> 静态页面是「能用」，流式推送+ML驱动是「顶级」。
> 每个模块让用户觉得「这个产品真懂我」。

---

## E1: 个人命理仪表盘

### 验收标准
- [ ] E1.1 GET /api/dashboard/{user_id} 返回完整仪表盘 JSON
- [ ] E1.2 包含：八字概览卡片 + 最近分析历史 + 准确率趋势 + 偏好总结
- [ ] E1.3 仪表盘风格匹配用户偏好（毒舌/分析/温柔）
- [ ] E1.4 首次用户返回友好引导（非空页面）
- [ ] E1.5 仪表盘数据实时聚合（来自 preference_dao + session_dao + dao）
- [ ] E1.6 API 响应 < 500ms（纯数据聚合，无 AI 调用）

---

## E2: 社交分享卡片

### 验收标准
- [ ] E2.1 用户可从八字分析结果生成分享卡片
- [ ] E2.2 卡片包含：AI 生成的个性化金句 + 五行图 + 关键数据
- [ ] E2.3 卡片尺寸 1080×1080（适合朋友圈/Instagram）
- [ ] E2.4 3 种视觉风格可选（暗黑/温暖/简约）
- [ ] E2.5 AI 金句由 Flash 动态生成（不硬编码）
- [ ] E2.6 卡片生成 < 5s（复用 chart cache）

---

## E3: 流式响应

### 验收标准
- [ ] E3.1 Pro 分析支持 SSE 流式输出（Server-Sent Events）
- [ ] E3.2 首字节 < 5s（用户看到「正在分析...」）
- [ ] E3.3 流式输出不阻塞（用户可以看到文字逐段出现）
- [ ] E3.4 图表生成与流式输出并行
- [ ] E3.5 流式输出失败时回退到完整响应模式

---

## E4: ML 响应质量预测

### 验收标准
- [ ] E4.1 基于历史反馈训练轻量质量预测模型
- [ ] E4.2 输入：用户消息特征 + 回复长度 + 人格模式 + 情绪标签
- [ ] E4.3 输出：预测该回复会被 👍 的概率
- [ ] E4.4 低质量预测 (<30%) → 自动重试或切换策略
- [ ] E4.5 模型使用在线学习（每次反馈后更新权重）
- [ ] E4.6 无需 GPU，纯 NumPy 逻辑回归
- [ ] E4.7 预测 < 10ms（不增加延迟）

### 技术方案
```
特征向量 (7维):
[msg_length, hour_of_day, personality_mode, emotion_label_id, 
 topic_id, response_length, has_birth_info]

→ 逻辑回归 → P(👍) ∈ [0, 1]

训练: 每次反馈后在线更新权重 (SGD, lr=0.01)
```

---

## 执行顺序

```
E1 (仪表盘) → E2 (分享卡片) → E3 (流式) → E4 (ML质量)
```

每个 E 完成后自测 + 部署验证。
